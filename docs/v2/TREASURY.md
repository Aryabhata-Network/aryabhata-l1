# Protocol Fee Treasury (PFT)

## Overview

Zero premine. Zero founder allocation.
Development funded through transparent fee accumulation.

---

## Accumulation

2% of every transaction fee routes automatically to PFT.
This happens at consensus layer.
Not configurable. Not bypassable. Not governable.
treasury_amount = tx_fee * 0.02
---

## Rules

| Rule | Specification |
|---|---|
| Time lock | All funds locked 90 epochs before withdrawal |
| Spending cap | Maximum 10% of balance per 90 epoch period |
| Multisig | 3-of-5 signatures required |
| Transparency | All transactions on-chain and labelled |
| Dissolution | If no block in 365 days, PFT is burned |

---

## Permitted Uses

- Protocol development
- Security audits
- Infrastructure costs
- Legal compliance

## Prohibited Uses

- Marketing or promotion
- Token buybacks
- Personal compensation above market rate
- Any use not listed above

---

## Key Holders

5 public keys stored in genesis block.
All key holders publicly identified.
No single key holder can access funds.
Minimum 3 of 5 must agree on every withdrawal.
